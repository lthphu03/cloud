Event 1

EVENT REPORT “FCAJ COMMUNITY DAY” - EVENT 1, MAY 23 ,2026

Event name: FCAJ Community Day

Main topics: AI, Amazon Quick, Amazon CloudFront, LLM, and Multi-Agent System

  • Learn how AI can work more effectively when proper context is provided.
  • Explore current trends in Generative AI, AI Assistants, and Multi-Agent Systems.
  • Understand the role of AWS services such as Amazon CloudFront, Amazon QuickSight/Quick Suite, and Amazon Bedrock in real-world use cases.
  • Listen to practical experiences in building products during hackathons and turning ideas into demos within a short time.
  • Gain a broader view of how AI, data, and cloud can be combined in enterprise environments.

Key Highlights

Context Is Everything: Making AI Actually Work for You

  • AI does not depend only on prompts but also heavily depends on the context provided.
  • Without proper context, AI may produce generic, inaccurate, or unsuitable results.
  • The Second AI Brain concept describes how AI can better remember, organize, and use personalized information.
  • Learners need to define problems clearly, provide background information, and evaluate AI-generated outputs.

Friendly AI Assistant with Amazon Quick

  • AI Assistants can help users explore data and analyze insights faster.
  • Users can create intelligent workflows using natural language without writing complex code.
  • Collaborative spaces can turn individual insights into shared team knowledge.
  • Creating dashboards and reports from raw data using natural language can reduce the time required for data analysis.

From Edge To Origin: CloudFront as Your Foundation

  • Amazon CloudFront helps deliver content from edge locations to users with lower latency.
  • CloudFront can support many types of workloads, not only static websites.
  • The service can help optimize cost, improve performance, and strengthen security.
  • CloudFront also plays an important role in improving system reliability when serving users across different regions.

36 hrs with LotusHacks – Building UTMorpho from Idea to Reality

  • Team VIB shared the process of building UTMorpho within 36 hours.
  • The team started from brainstorming, problem definition, solution shaping, and demo implementation.
  • The hackathon experience showed the importance of teamwork, time management, and fast decision-making.
  • The sharing helped participants understand how an initial idea can become a working prototype.

Non-Determinism of “Deterministic” LLM Settings

  • Temperature = 0 does not absolutely guarantee that an LLM will always return the same output.
  • Inference processes, system optimizations, and token selection may still create small differences in results.
  • This behavior can affect systems that require high consistency.
  • When applying LLMs in real-world systems, testing, logging, and output control strategies are important.

Enterprise-Grade Multi-Agent System: Startup Credit Scoring

  • Startup credit scoring is complex because startup data is different from traditional financial data.
  • A Multi-Agent System can simulate different roles in a decision-making process.
  • The Virtual Credit Committee model helps organize evaluation from multiple perspectives.
  • Guardrails, compliance, and ROI are important considerations when deploying AI in enterprise environments.

What I Learned

AI Thinking

  • Understood that AI needs clear context to generate better results.
  • Learned that prompts are only one part of the process; background data and problem description are also important.
  • Realized that AI should not be used mechanically and still requires human review and evaluation.

AWS Cloud

  • Learned more about the role of Amazon CloudFront in content delivery, performance optimization, and security.
  • Understood how AWS services can support data analysis, dashboard creation, and system deployment.
  • Gained a broader view of combining AI with cloud services to solve real-world problems.

Product Development

  • Learned how a team can quickly develop a product in a hackathon environment.
  • Understood that defining the right problem is as important as building the solution.
  • Realized that teamwork, task division, and product demo skills are important in software development.

Enterprise AI Systems

  • Understood that LLMs may still be non-deterministic even when the configuration appears deterministic.
  • Learned about Multi-Agent Systems and how multiple agents can collaborate to solve a complex problem.
  • Realized that guardrails, compliance, and risk control are important when deploying AI at enterprise scale.

Application to Work

  • Apply clearer prompt-writing methods when using AI to support learning, documentation, and information analysis.
  • Use AI to suggest insights, support dashboard creation, or assist with initial data processing tasks.
  • Pay more attention to performance and security-related AWS services such as CloudFront when learning AWS.
  • When building personal projects, start from problem definition, target users, and product value.
  • When using AI in real systems, include result validation, logging, and scope limitation.

Event Experience

Joining this event gave me a more practical perspective on how AI and AWS Cloud are being applied in different fields. The sessions were not only theoretical but also included practical examples such as building a product during a hackathon, optimizing systems with CloudFront, and applying Multi-Agent Systems to credit scoring.

The session about context in AI helped me understand that using AI effectively requires clear information and a specific goal. The LLM session also helped me realize that modern AI systems still need careful control, especially in use cases that require accuracy and stability.

Networking and Discussion

  • Had the opportunity to learn from speakers working in AI, Cloud, and Data fields.
  • Gained more insight into career directions related to AI, cloud computing, and data analysis.
  • Collected useful ideas that can be applied to learning, internship tasks, and personal projects.

Key Takeaways

  • Context is a key factor that helps AI generate relevant and valuable results.
  • AWS Cloud provides many services that support infrastructure, content delivery, AI, and data analytics.
  • It is important not only to learn individual tools but also to understand how services work together to solve real-world problems.
  • When deploying AI in real environments, stability, security, risk control, and business value must be considered.
  • The event gave me more motivation to continue learning AWS, AI, and building cloud-based applications in the future.
