Event 2

EVENT REPORT “SATURDAY MEET UP - JUNE 06, 2026”

Event Objectives

  • Learn practical applications of AWS in different areas such as gaming, system administration, data, DevOps, and cybersecurity.
  • Expand cloud computing knowledge through topics such as WebSocket, Serverless, Graph Database, Docker, and AWS WAF.
  • Listen to career experience from IT Helpdesk to Senior Sysadmin and the transition toward Cloud/DevOps.
  • Understand the role of teamwork in learning, project development, and product building.
  • Learn how AI and Machine Learning can be applied in GraphRAG and cyber attack detection.

Key Highlights

Multiplayer in the Cloud: Connecting Godot Clients with AWS WebSockets

  • The session introduced how to build a simple multiplayer system on the cloud.
  • Godot clients can connect to the backend through WebSocket.
  • AWS API Gateway WebSocket is used to maintain connections between clients and the backend.
  • AWS Lambda handles logic such as player connection, matchmaking, and game results.
  • DynamoDB is used to store player states such as connection ID, waiting status, and opponent information.
  • The content helped participants understand how AWS can support basic realtime applications.

From IT Helpdesk to Senior Sysadmin

  • The session focused on the career development path in IT Infrastructure.
  • IT Helpdesk is a good foundation for developing troubleshooting skills and user communication.
  • To become a Sysadmin, learners need to build knowledge in Linux, Networking, Servers, Monitoring, and Security.
  • When transitioning to Cloud/DevOps, learners need to become familiar with automation, Infrastructure as Code, CI/CD, Docker, and cloud platforms.
  • An important lesson is not to learn too many topics at the same time, but to build a strong foundation first.

The Art of Effective Teamwork

  • Effective teamwork requires clear goals that are shared among all members.
  • Each member should be assigned to the right role based on skills and strengths.
  • Open communication, active listening, and clear feedback help reduce misunderstandings during teamwork.
  • Personal accountability is important to maintain overall team progress.
  • Some useful tools for teamwork include Trello, ClickUp, Google Workspace, Slack, and Discord.

AWS Neptune for Building a Graph Knowledge Base for GraphRAG

  • The session introduced GraphRAG, an extension of Retrieval-Augmented Generation.
  • Traditional RAG helps LLMs retrieve information from external sources but may be limited when multi-hop reasoning is required.
  • GraphRAG uses a graph database to represent relationships between entities.
  • Amazon Neptune can be used to build a graph knowledge base.
  • GraphRAG is suitable for use cases that require understanding complex relationships, such as intelligent Q&A systems or enterprise knowledge bases.
  • The topic also mentioned combining Amazon Bedrock, Neptune Analytics, and data processing pipelines to support smarter retrieval.

Docker - A Containerization Technology

  • Docker packages applications together with the runtime environment and required dependencies.
  • Containers are lighter than virtual machines because each application does not need a separate operating system.
  • Docker supports the idea of “build once, run anywhere”.
  • Important Docker concepts include Docker Image, Docker Container, Dockerfile, Layer, and Docker commands.
  • Docker is commonly used in CI/CD, microservices, development/testing environments, and cloud-native applications.
  • The session helped explain why Docker is an important foundation for Cloud and DevOps.

Combining AWS WAF with Machine Learning for Cyber Attack Detection on AWS

  • AWS WAF helps protect web applications from common attacks such as SQL Injection, XSS, bot traffic, and abnormal requests.
  • Rule-based WAF protection is effective for known attack patterns but may be limited against new types of attacks.
  • Machine Learning can help detect abnormal behavior based on network data.
  • The session mentioned dataset processing, data cleaning, model training, and result evaluation.
  • The AWS architecture may combine services such as WAF, EC2, ALB, S3, Lambda, CloudWatch, GuardDuty, Security Hub, and SNS.
  • The content helped participants understand the combination of Cloud Security and Machine Learning.

What I Learned

AWS Cloud

  • AWS is not only used for creating servers or storing data, but can also support realtime games, AI, security, and distributed systems.
  • Services such as API Gateway, Lambda, and DynamoDB can be combined to create a simple serverless backend.
  • Amazon Neptune supports graph-based data storage, which is suitable for relationship-heavy problems.
  • AWS WAF, CloudWatch, GuardDuty, and Security Hub are important services when building security-aware systems.

DevOps and Systems

  • Docker is an important tool that helps applications run consistently across different environments.
  • To follow a Cloud/DevOps path, learners need foundations in Linux, Networking, System Administration, Automation, and CI/CD.
  • Learning cloud effectively requires a combination of theory, hands-on labs, and real projects.
  • When deploying systems, monitoring, security, cost, and scalability should be considered.

AI and Data

  • GraphRAG helps LLMs better handle questions that require understanding relationships between multiple entities.
  • Machine Learning can support cyber attack detection by identifying abnormal behaviors.
  • Data quality is very important in AI/ML problems.
  • AI should not be treated as a standalone tool, but should be combined with data, infrastructure, and proper control processes.

Teamwork and Career Direction

  • Effective teamwork requires shared goals, clear task assignment, and personal accountability.
  • Communication and listening skills are as important as technical skills.
  • The path from IT Helpdesk to Sysadmin and Cloud/DevOps requires time, practice, and a strong foundation.
  • A practical portfolio and real projects are valuable for career development.

Application to Work

  • When learning AWS, I can try combining several small services to understand how a cloud system works end-to-end.
  • Docker can be applied to package personal projects, making them easier to deploy and test.
  • In team projects, tasks should be clearly assigned, progress should be updated regularly, and goals should be aligned from the beginning.
  • For AI-related projects, it is important to pay attention to data sources, data organization, and result evaluation.
  • When learning cloud security, multiple protection layers such as WAF, logging, monitoring, and alerting should be considered.
  • Knowledge of Sysadmin, DevOps, and Cloud can support my future career direction after graduation.

Event Experience

The event on June 6 provided a wide range of topics, not focusing on only one AWS service but expanding into realtime games, DevOps, AI, GraphRAG, Docker, and Cybersecurity. The sessions helped me see more clearly how AWS can be applied in real situations, from game backend development and knowledge base construction to cyber attack detection.

One useful point was that the topics were closely related to the learning path of IT students. For example, the session from IT Helpdesk to Sysadmin helped me understand the foundation needed before learning Cloud/DevOps. The Docker session helped me realize that containerization is an important skill for modern application deployment. The GraphRAG and AWS WAF with Machine Learning sessions also expanded my view of how AI can be applied in data and security.

Networking and Discussion

  • Had the opportunity to access multiple technical topics in one event.
  • Learned practical experience and career direction from the speakers.
  • Gained ideas that can be applied to AWS learning, personal projects, and Cloud/DevOps preparation.
  • Realized that technical skills should go together with teamwork, communication, and problem-solving mindset.

Key Takeaways

  • AWS is a broad ecosystem that can support many types of applications, from web, games, and data to AI and security.
  • To learn Cloud effectively, it is important to understand fundamentals such as networking, systems, databases, security, and deployment.
  • Docker is an important skill for Cloud Native and DevOps directions.
  • AI/ML can create significant value when combined properly with data and cloud infrastructure.
  • Security should not rely only on fixed rules, but also needs monitoring, logging, and smarter detection methods.
  • Good teamwork helps projects run more effectively and reduces risks during development.
