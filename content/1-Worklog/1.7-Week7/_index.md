### Week 7 Objectives:

* Learn about AWS Command Line Interface (AWS CLI) and its role in managing AWS resources.
* Become familiar with managing AWS through command line instead of only using AWS Management Console.
* Learn about AWS CloudShell/Cloud9 as an environment for practicing and running AWS commands.
* Practice several basic AWS CLI commands to check account information, Region, and resources.

### Week 7 Achievements:

* Understood that AWS CLI is a tool that allows users to manage and interact with AWS resources through command line.

* Learned the basic difference between AWS Management Console and AWS CLI:

  * AWS Console is suitable for visual operations
  * AWS CLI is useful for quick checks, automation, and command-based management

* Learned how to check the personal computer environment before using AWS CLI.

* Learned how to install and check the AWS CLI version.

* Understood the important information required when configuring AWS CLI:

  * Access Key ID
  * Secret Access Key
  * Default Region
  * Output Format
  * Profile

* Practiced several basic AWS CLI commands to check configuration and resources.

* Learned how to use commands to check AWS account identity at a basic level.

* Became familiar with querying EC2 and S3 information through command line.

* Learned about AWS CloudShell/Cloud9 as an environment that supports running AWS commands directly on the cloud platform.

* Realized that AWS CLI helps speed up tasks when checking resources or performing repeated operations.

* Became more careful when using Access Key and Secret Access Key because they are sensitive information related to AWS access permissions.

* Completed Week 7 Worklog and updated the content on the personal Hugo website.
