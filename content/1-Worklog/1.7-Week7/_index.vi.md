### Mục tiêu tuần 7:

* Tìm hiểu AWS Command Line Interface (AWS CLI) và vai trò của CLI trong việc quản lý tài nguyên AWS.
* Làm quen với việc thao tác AWS thông qua dòng lệnh thay vì chỉ sử dụng AWS Management Console.
* Tìm hiểu AWS CloudShell/Cloud9 như môi trường hỗ trợ thực hành và chạy lệnh AWS.
* Thực hành một số lệnh AWS CLI cơ bản để kiểm tra tài khoản, Region và tài nguyên.

### Kết quả đạt được tuần 7:

* Hiểu được AWS CLI là công cụ cho phép quản lý và thao tác với tài nguyên AWS thông qua dòng lệnh.

* Nắm được sự khác nhau cơ bản giữa việc sử dụng AWS Management Console và AWS CLI:

  * AWS Console phù hợp để thao tác trực quan
  * AWS CLI phù hợp để kiểm tra nhanh, tự động hóa và quản lý bằng lệnh

* Biết cách kiểm tra môi trường máy tính cá nhân trước khi sử dụng AWS CLI.

* Tìm hiểu được cách cài đặt và kiểm tra phiên bản AWS CLI.

* Hiểu được các thông tin quan trọng khi cấu hình AWS CLI:

  * Access Key ID
  * Secret Access Key
  * Default Region
  * Output Format
  * Profile

* Thực hành một số lệnh AWS CLI cơ bản để kiểm tra cấu hình và tài nguyên.

* Biết cách sử dụng lệnh để kiểm tra danh tính tài khoản AWS ở mức cơ bản.

* Làm quen với việc truy vấn thông tin EC2 và S3 thông qua dòng lệnh.

* Tìm hiểu AWS CloudShell/Cloud9 như môi trường hỗ trợ chạy lệnh AWS trực tiếp trên nền tảng cloud.

* Nhận thấy AWS CLI giúp thao tác nhanh hơn khi cần kiểm tra tài nguyên hoặc thực hiện các tác vụ lặp lại.

* Cẩn thận hơn trong việc sử dụng Access Key và Secret Access Key vì đây là thông tin nhạy cảm liên quan đến quyền truy cập AWS.
