### Week 6 Objectives:

* Learn about Amazon Relational Database Service (Amazon RDS).
* Understand the role of relational databases in cloud-based systems.
* Become familiar with popular Database Engines such as MySQL, PostgreSQL, and Amazon Aurora.
* Learn the basic settings used when deploying RDS, such as Instance Class, Storage, Backup, Security Group, and VPC.

### Week 6 Achievements:

* Understood that Amazon RDS is a managed relational database service provided by AWS.

* Learned the benefits of RDS compared with manually installing and managing a database on EC2.

* Understood the common Database Engines supported by RDS:

  * MySQL
  * PostgreSQL
  * MariaDB
  * Microsoft SQL Server
  * Oracle Database
  * Amazon Aurora

* Became familiar with Amazon Aurora and understood that it is a relational database service optimized for the AWS environment.

* Learned the important configuration information when setting up an RDS Database:

  * DB Engine
  * DB Instance Class
  * Storage
  * Username and Password
  * VPC
  * Security Group
  * Backup

* Understood the role of Security Group in controlling connections to a database.

* Distinguished between Public Access and Private Access at a basic level when configuring a database on AWS.

* Learned the meaning of Backup, Snapshot, and Multi-AZ in protecting data and improving availability.

* Learned how to access the Amazon RDS and Amazon Aurora sections on AWS Management Console to check resource status.

* Became aware that RDS may generate costs if it is configured incorrectly or if resources are not removed after practice.

* Understood the need to check the Billing Dashboard and resource status after exploring Database services.

* Completed Week 6 Worklog and updated the content on the personal Hugo website.
