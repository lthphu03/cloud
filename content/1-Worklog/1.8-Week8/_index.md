### Week 8 Objectives:

* Learn about Amazon CloudWatch and its role in monitoring AWS resources.
* Understand basic concepts such as Metrics, Logs, Alarms, and Dashboards.
* Become familiar with AWS CloudTrail for tracking activity history in the AWS account.
* Learn how to observe resource status and review events related to the AWS account.

### Week 8 Achievements:

* Understood that Amazon CloudWatch is a service used to monitor AWS resources and applications.

* Learned the basic components of CloudWatch:

  * Metrics
  * Logs
  * Alarms
  * Dashboards

* Learned how to access the CloudWatch interface from AWS Management Console.

* Understood that Metrics are measurement data used to represent the operational status of AWS resources over time.

* Became familiar with CloudWatch Logs and understood how logs are organized through Log Groups and Log Streams.

* Understood that CloudWatch Alarms can be used to send alerts when a resource exceeds a defined threshold.

* Became aware of the importance of monitoring resources after practicing services such as EC2, Lambda, RDS, or S3.

* Learned about AWS CloudTrail and its role in recording activity history in an AWS account.

* Learned how to view Event History to check actions performed in the account.

* Distinguished the basic roles of CloudWatch and CloudTrail:

  * CloudWatch supports performance monitoring, logs, and alarms
  * CloudTrail supports API activity tracking and account auditing

* Understood that monitoring and auditing are important parts of managing AWS resources safely and effectively.

* Completed Week 8 Worklog and updated the content on the personal Hugo website.
