### Week 2 Objectives:

* Learn about AWS Identity and Access Management (IAM).
* Understand how identity management and access control work on AWS.
* Become familiar with Users, Groups, Roles, and Policies.
* Practice managing access permissions based on basic security principles.

### Week 2 Achievements:

* Understood the role of AWS IAM in managing identities and controlling access on the AWS platform.

* Learned the functions of the main IAM components:

  * IAM User
  * IAM Group
  * IAM Role
  * IAM Policy

* Understood the difference between Root User and IAM User, and why Root User should be limited in daily administration tasks.

* Became familiar with the IAM interface on AWS Management Console and learned how to find user management features.

* Learned how AWS Managed Policies help assign access permissions quickly to users and groups.

* Learned the **Least Privilege** principle, which helps grant only the necessary permissions to each user or service.

* Understood the role of MFA in improving AWS account security.

* Gained the ability to read and understand basic Policies for future AWS service learning.

* Summarized important notes when managing accounts and permissions on AWS.

* Completed Week 2 Worklog and updated the content on the personal Hugo website.
