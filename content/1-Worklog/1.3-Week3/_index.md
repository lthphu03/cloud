### Week 3 Objectives:

* Learn about Amazon EC2 and its role in the Compute category on AWS.
* Understand the basic components required when launching an EC2 Instance, such as AMI, Instance Type, Key Pair, Security Group, and EBS.
* Practice creating a basic EC2 Instance within the Free Tier scope.
* Learn how to connect to an EC2 Instance using SSH and check the resource status after practice.

### Week 3 Achievements:

* Understood that Amazon EC2 is a service that provides virtual servers on AWS and belongs to the Compute service category.

* Learned the role of EC2 in deploying applications, testing server environments, and building foundational cloud infrastructure knowledge.

* Understood the important components used when launching an EC2 Instance:

  * Amazon Machine Image (AMI)
  * Instance Type
  * Key Pair
  * Security Group
  * Elastic Block Store (EBS)

* Learned how to choose a suitable Instance Type for learning purposes, especially Free Tier eligible options.

* Practiced creating a basic EC2 Instance using AWS Management Console.

* Understood the role of Security Group in controlling inbound and outbound traffic for an EC2 Instance.

* Learned how to check the Public IPv4 Address before connecting to EC2.

* Learned how to connect to an EC2 Instance using SSH and a Key Pair.

* Observed the Instance running status on AWS Console and learned how to check resources by Region.

* Understood that EBS is a block storage service commonly used with EC2 Instances.

* Learned how to stop or delete resources after practice to reduce the risk of unnecessary charges.

* Completed Week 3 Worklog and updated the content on the personal Hugo website.
